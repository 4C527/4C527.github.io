最简单的设计模式：单例模式

       本篇博客介绍设计模式中最简单的一种：单例模式，不过虽然简单，但它仍然在工业界发挥着巨大的作用，这是一种小而美的设计模式。单例模式的作用很简单，它的目的就是确保某个类只有一个实例，而且自行实例化并向整个系统提供这个实例。在计算机系统中，线程池、缓存、日志对象、对话框、打印机、显卡的驱动程序等对象常被设计成单例。举个栗子，每台计算机可以有若干个打印机，但只能有一个Printer Spooler，以避免两个打印作业同时输出到打印机中。总之，选择单例模式就是为了避免出现不一致状态，这在软件开发中有广泛的用处。
       单例模式有以下特点：
       1、单例类只能有一个实例。
       2、单例类必须自己创建自己的唯一实例。
       3、单例类必须给所有其他对象提供这一实例。
