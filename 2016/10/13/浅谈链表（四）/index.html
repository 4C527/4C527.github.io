判断有环链表环开始节点

       继判断一个单链表是否有环这个算法之后，它的升级问题也是一般公司面试时候常问的：如何找到一个有环链表开始的那个节点？
       leetcode原题如下：


       这道题的大意就是判断一个链表是否有环，如果有环，返回环开始的节点，如果没有环，则返回null.
       这题的暴力解法这里就不再讨论，基本上在leetcode当中很少有算法题能通过穷举法得到答案。我们只提出一个好的，高效的算法。
       其实这个算法的代码最终答案很简单：如果有环，那快慢指针会在环上相遇。现在令一个指针指向头节点，一个指针指向相遇节点，两个指针同时走，则它们一定会相遇，而且相遇的点就是环开始的点。
       那可能有人会说，为什么一定这两个指针相遇点就是环开始点？没关系，接下来我们就通过数学证明的方式证明它们相遇的点必然是环开始的节点。
       证明过程如下：
       设非环部分有x个节点，环开头到快慢指针相遇点有y个节点，快慢指针相遇点重回环开头有z个节点。如下图所示：


       再设快指针走到相遇点时共经历了a圈环（即在环上走了a遍），则总共经历的节点数目为:x + a(y + z) + y;
       设慢指针走到相遇点时共经历了b圈环（即在环上走了b遍），则总共经历的节点数目为：x + b(y + z) + y;
       由于快指针一次走2步（即走2个节点），慢指针一次走1步，所以等式：x + a(y + z) + y = 2[x + b(y + z) + y];
       等式两边同时加上一个z,即：x + a(y + z) + y + z = 2[x + b(y + z) + y] + z;
       两边消去一部分， 得：a(y + z) + z = x + y + 2b(y + z) + z;
       再次消去一部分：x = (a - 2b - 1) (y + z) + z;
       由于a,b都是整数，且y + z就等于环上节点数目，所以x就等于z加上任意整数倍个环上节点数目。
       所以得证，现在令一个指针i指向头节点，一个指针j指向相遇节点，两个指针同时走，i指针走了x步，j指针一定走了z步 + 任意圈环，则它们相遇的点就是环开头。
