第五章 总结与展望

5.1 总结

       TensorFlow开源至今已有大半年的时间（截止本报告完成前），也从之前的0.1单机版本更新到了支持分布式机器学习的0.8版本。下面，就TensorFlow的优缺点给出本调研报告的总结。

5.1.1 TensorFlow优点

       TensorFlow开源至今已有大半年的时间（截止本报告完成前），也从之前的0.1单机版本更新到了支持分布式机器学习的0.8版本。下面，就TensorFlow的优缺点给出本调研报告总结。
       一、社区活跃度高
       TensorFlow社区的大多数人都是开放心态，并对优化TensorFlow这一开源项目满怀热情。谷歌Deepmind团队也与近期宣布从Torch迁移到TensorFlow，因此谷歌很可能在未来会开源TensorFlow RL（强化学习）模型。当社区的每个人都具有开放性、愿意贡献API 、有用的模块并且乐于助人时，TensorFlow社区的未来一定是光明的。

       二、技术障碍基本扫清
       2015年11月份，TensorFlow刚刚开源时，还存在一些真正的和潜在的技术问题。但是目前这些基本上都已经被扫清，比如：
       1、支持多GPU：已经完成，在官网文档中已有教程，清晰简洁。
       2、分布式训练：v0.8已经支持分布式训练。
       3、支持操作队列：数据加载或图像预处理。
       4、TensorBoard支持图形可视化：当编程人员学习和调试新模型时，容易陷入迷惑，因此TensorBoard图形可视化非常有用。
       5、使用TensorBoard以交互方式记录日志：通过TensorBoard可以随时进行日志记录，比如学习速率、损失值、测试精度等。
       6、模型检查点设置：一个模型训练一段时间后，编程人员可以按需求于检查点停下来进行评估，然后从检查点重新加载，继续训练。
       7、使用CUDNN优化GPU性能：与Theano等框架类似，使用了CUDNN进行GPU性能优化。TensorFlow v0.8目前使用 CUDNNv4，性能有所提高。

       三、有几个高质量的元框架(metaframework)
       1、Keras封装了TensorFlow 和Theano后端。对于不想了解TensorFlow (或者Theano)的使用者来说，只需要模块化地调用它。
       2、TensorFlow Slim是对图像模型来说不错的参考。Slim 对开发者使用TensorFlow API和模型设计都有极大的参考价值。
       3、Scikit Flow 是 TensorFlow 的简化接口，是一个参考 Scikit-learn风格的API。
       4、PrettyTensor提供类似张量（tensors）的对象，并有可链接的语法，可以构建某种类型的模型。

       四、完善的版本进度管理
       维护一个开源项目，尤其是TensorFlow这种技术复杂的项目挑战非常大。TensorFlow团队的运营人员对于版本管理细节做的非常完善。

       五、测试出色
       测试对功能验证来说非常重要，当用户发现TensorFlow训练情况与预期的不一样时，用户可以在Github搜索TensorFlow测试了解TensorFlow测试方法，并进行测试，观察测试结果。

5.1.2 TensorFlow缺点

       一、相比Theano框架，在RNN实现方面还存在不足
       RNN(Recurrent Neural Networks)循环神经网络，属于Recursive Neural Network（递归神经网络的一种变体）。Recurrent指的是时间维度的展开，代表信息在时间维度从前往后的的传递和积累，后面的信息的概率建立在前面信息的基础上，在神经网络结构上表现为后面的神经网络的隐藏层的输入是前面的神经网络的隐藏层的输出，而recursive指的空间维度的展开，是一个树结构。
Theano 团队在优化他们的循环神经网络实现付出了很多的精力，在这方面TensorFlow还有较大的差距，但是近几个月这种差距在缩小。
       对于循环神经网络，在单 GPU 的情形下，Theano 可能比 TensorFlow 快一些，而且占用 GPU 内存小一点。但是在多 GPU 方面TensorFlow略占优势。

       二、数据训练目前还缺乏令人信服的示例
       TensorFlow 文档和案例主要使用一些著名的学术数据集来阐述各种特性或功能。这是有一定道理的，但是，现实中的问题几乎不能简单替换这些数据集。
开发者在学习TensorFlow 时最困难的是，如何将现实中杂乱的数据抽象为模型能识别的各种张量（tensor），因此如果有一些令人信服的案例，具体展示如何完成抽象过程，会对开发者来说非常有帮助。遗憾的是，谷歌并没有提供更多的这方面的示例。因此作为开发人员，往往需要在将显示数据抽象为模型输入张量这一具体过程中花费很多时间。

       三、文档尚不完善
       尽管TensorFlow 公布了一些教程，其代码注释的质量很高，但是由于深度学习领域范围广，发展很快，在新功能与教程之间还有很大的不足。

       四、TensorFlow 独占GPU
       在初始化时， TensorFlow 尝试去分配所有可用的GPU内存给自己。这是一把双刃剑，取决于使用者的情景。如果使用者开发一个模型，使用的是本地计算机上的 GPU，那么使用者可能会想分配一部分 GPU 到其他的事上。如果使用者在云计算环境下运行模型，那么就需要了解有没有其他相互影响的代码。

       五、默认情况下，Theano 与TensorFlow会相互冲突
       如果使用者在相同资源下运行 Theano 和 TensorFlow ，它们将会争夺 GPU 内存。为了在不同的 GPU 上运行完全不同的环境（比如两个 GPU 运行两种模式），用户可以限制 CUDA 只能探查到某种特定的设备。 CUDA是一种由NVIDIA推出的通用并行计算架构，该架构使GPU能够解决复杂的计算问题。它包含了CUDA指令集架构(ISA)以及GPU内部的并行计算引擎。

       六、其他
       与其他学习框架相比，一些深度学习的基本概念是通用的，但是实现细节区别很大。比如队列（queues）, 特定图形运算（certain graph operations）、资源配置/内容管理（ resource allocation/context management）, 图像可视化（graph visualization）这些细节对很多深度学习开发者来说都还比较陌生，当然，所有TensorFlow的使用者也在不断学习。

5.2 展望

       当开源社区内的一些人实现了一次很棒的训练或者尝试了一些解决问题的新方法时，开源工具的优势就显现出来了。即使大部分人现在仍然处于学习使用 TensorFlow 的初级阶段，但是已经不断有开发者取得不错的进展！让我们展望人工智能的新时代！
